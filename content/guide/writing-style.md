# aieazy 블로그 글 작성 스타일 가이드

## 목표
AI가 쓴 것 같은 설명 톤을 줄이고, 실제 블로거가 직접 확인하고 정리한 듯한 **자연스러운 한국어 글투**로 작성한다.

다만 자연스러움 때문에 정보가 흐려지면 안 된다.  
**말투는 편하게, 결론과 절차는 정확하게** 쓰는 것이 기준이다.

---

## 핵심 원칙

### 1. 검색 의도 먼저 답하기
- 글의 첫 부분에서 독자가 찾는 답을 바로 보여준다.
- 가격 글은 가격표나 추천 기준부터, 오류 해결 글은 해결 순서부터, 프롬프트 글은 복사 예시부터 시작한다.
- 정의나 배경 설명은 필요한 경우에만 뒤쪽에 둔다.

예시:

```md
ChatGPT 결제가 안 될 때는 먼저 카드 승인 상태와 결제 국가를 확인해보세요.
대부분은 카드사 승인 거절, VPN, 계정 국가 문제에서 막힙니다.

## 결제 안 될 때 먼저 확인할 것
```

### 2. 문장 구조
- 단락은 기본적으로 **1~2문장**으로 쓴다.
- 한 문단에는 **핵심 1개만** 담는다.
- 짧은 문장을 기본으로 하되, 설명이 필요한 문장은 40~70자까지 허용한다.
- `~기 때문에`, `~하는 경우에는` 같은 복문은 너무 자주 쓰지 않는다.

짧게 끊는 것보다 중요한 건 읽는 흐름이다.  
문장이 너무 잘게 쪼개져서 기계적으로 보이면 한 문장으로 합친다.

### 3. 말투
- 기본 톤은 **친근한 전문가**다.
- `~해보세요`, `~해주세요`, `~하면 됩니다` 같은 행동형 종결을 자연스럽게 섞는다.
- `~는데요`, `~거든요`, `~더라고요` 같은 구어체는 허용한다.
- `~입니다/합니다` 체만 반복하지 않는다.
- 사용법, 가격, 환불, 오류 해결처럼 정확성이 중요한 부분은 단정적으로 쓴다.

좋은 방향:
- "먼저 설정 메뉴에서 결제 정보를 확인해보세요."
- "카드 승인 거절이면 카드사 앱에서 실패 내역이 보이는 경우가 많아요."
- "환불 가능 여부는 결제 경로에 따라 달라집니다."

피해야 할 방향:
- "아마 이런 것 같아요."
- "대충 설정에서 찾아보면 됩니다."
- "정확하진 않지만 대부분 될 거예요."

### 4. AI 느낌을 주는 패턴 줄이기

| 피할 표현 | 이유 | 자연스러운 대체 |
| --- | --- | --- |
| "~하는 방법을 알려드리겠습니다" | AI 설명 톤 | "~하는 방법을 정리해봤어요" |
| "다음과 같습니다" | 딱딱한 목록 나열 | "이렇게 보면 됩니다" |
| "중요한 것은 ~입니다" | 강조가 과함 | "~하는 게 핵심이에요" |
| "한 번 따라해보세요" 반복 | 패턴화된 유도 | 문맥에 맞게 바로 행동 지시 |
| "~할 수 있습니다" 반복 | 수동적이고 건조함 | "~하면 돼요", "~에서 가능합니다" |
| "확인하시기 바랍니다" | 공지문 느낌 | "확인해보세요" |
| "모든 ~를" | 과도한 포괄 | "대부분", "주요", "자주 쓰는" |

단, `정확히`, `반드시`, `해야 합니다` 같은 표현은 완전 금지가 아니다.  
결제, 보안, 환불, 계정 삭제, API 키 입력처럼 실수가 손해로 이어지는 부분에서는 필요하면 쓴다.

### 5. 반복 패턴 피하기
- 같은 문장 구조를 여러 번 반복하지 않는다.
- `~해주세요 -> ~해주세요 -> ~해주세요`처럼 같은 종결을 3번 연속 쓰지 않는다.
- 단계별 글에서도 말투를 조금씩 바꾼다.

예시:
- "눌러주세요"
- "목록에서 선택합니다"
- "마지막으로 저장됐는지 확인해보세요"

### 6. 자연스러운 생략과 정보 정확도
- 감정, 배경, 사소한 부연 설명은 생략해도 된다.
- 클릭 경로, 설정명, 결제 조건, 오류 메시지, 주의사항은 생략하지 않는다.
- 독자가 따라 하다가 막힐 만한 부분은 짧게라도 적는다.

나쁜 예:
> 설정에서 찾아보세요.

좋은 예:
> 왼쪽 메뉴의 설정에서 찾을 수 있어요. 화면이 다르면 검색창에 설정명을 입력해보세요.

### 7. 개인 경험과 검증 흔적 넣기
- 실제로 확인한 내용은 1인칭 경험으로 자연스럽게 적는다.
- 직접 확인하지 않은 내용을 경험담처럼 쓰지 않는다.
- 경험담이 없을 때는 화면 기준, 공식 안내 기준, 테스트 환경 기준으로 표현한다.

사용 가능한 표현:
- "저도 처음에 여기서 조금 헤맸는데요."
- "제가 직접 해보니 저장 버튼이 아래쪽에 있더라고요."
- "공식 도움말과 실제 화면을 같이 보면 이 순서가 가장 안전합니다."
- "테스트 환경에서는 이렇게 표시됐어요."

피해야 할 표현:
- 확인하지 않았는데 "제가 해보니"라고 쓰기
- 근거 없이 "대부분 다 됩니다"라고 단정하기
- 개인 느낌만으로 가격, 환불, 정책을 설명하기

### 8. 불완전한 표현은 조심해서 쓰기
- `대부분`, `가끔`, `브라우저마다 다를 수 있어요` 같은 표현은 자연스럽다.
- 다만 해결 글에서는 예외가 생겼을 때 무엇을 확인할지 같이 적는다.

예시:
> 대부분은 캐시 삭제 후 다시 로그인하면 해결돼요. 그래도 안 되면 시크릿 창에서 한 번 더 확인해보세요.

### 9. 강조 표현 제한
- 노란 하이라이트는 한 문단에 1번만 사용한다.
- 빨간색 강조는 글 전체에서 2~3번 이내로 제한한다.
- 강조가 많으면 AI가 임의로 중요도를 붙인 느낌이 난다.
- 자연스러운 문장 강조는 `핵심은 ~예요`, `여기서 많이 막힙니다`처럼 쓴다.

### 10. 이미지 설명 스타일
- 이미지 설명은 짧고 구체적으로 쓴다.
- `[이미지: 화면 캡처]`처럼 너무 추상적으로 쓰지 않는다.
- 이미지 설명도 1문장으로 제한한다.

좋은 예:
- `[이미지: Clarity 대시보드에서 새 프로젝트 버튼이 보이는 화면]`
- `[이미지: ChatGPT 결제 관리 메뉴 위치]`

## 주제별 톤 조정

### 사용법 글
- 첫 섹션에서 전체 순서를 먼저 보여준다.
- 단계마다 클릭 위치와 메뉴명을 생략하지 않는다.
- 말투는 가볍게, 절차는 정확하게 쓴다.

### 오류 해결 글
- 원인 설명보다 해결 순서를 먼저 둔다.
- 가장 가능성 높은 방법부터 정렬한다.
- 안 됐을 때 다음에 확인할 항목을 남긴다.

### 가격, 환불, 취소 글
- 조건, 예외, 결제 경로를 정확하게 쓴다.
- `~같아요`보다 `~입니다`, `~에 따라 달라집니다`를 사용한다.
- 정책이 바뀔 수 있는 내용은 확인 날짜나 공식 페이지 기준을 적는다.

### 비교 글
- 초반에 요약 표와 추천 기준을 먼저 둔다.
- 장단점은 과장하지 않는다.
- 특정 도구를 추천할 때는 누구에게 맞는지도 함께 쓴다.

### 프롬프트, 템플릿 글
- 복사해서 쓸 수 있는 예시를 위쪽에 둔다.
- 설명은 예시 뒤에 붙인다.
- 프롬프트가 어떤 상황에 맞는지 짧게 적는다.

## 금지에 가까운 표현

반복 사용하면 AI 느낌이 강해진다.

- "알려드리겠습니다"
- "다음과 같습니다"
- "중요한 것은"
- "확인하시기 바랍니다"
- "본 글에서는"
- "이를 통해"
- "효율적으로 활용할 수 있습니다"
- "다양한 기능을 제공합니다"
- "사용자 경험을 향상시킵니다"

## 제한해서 쓸 표현

- `~해보세요`: 한 문단에 1회 정도
- `~하면 됩니다`: 한 문단에 1회 정도
- `~입니다`: 연속 2회까지만
- `정확히`: 입력값, 금액, 날짜, 정책처럼 정확성이 필요한 곳에서만
- `반드시`: 보안, 삭제, 환불, 결제처럼 위험이 있는 곳에서만
- `~같아요`: 개인 경험이나 느낌에만 사용

## 권장 표현

### 행동형
- "눌러보세요"
- "선택해주세요"
- "확인해보세요"
- "들어가보세요"
- "저장됐는지 봐주세요"
- "아래쪽까지 내려가보세요"

### 경험형
- "저도 여기서 한 번 막혔어요"
- "보통 이 메뉴에서 찾을 수 있어요"
- "은근히 아래쪽에 숨어 있더라고요"
- "생각보다 간단한데 위치가 헷갈릴 수 있어요"

### 신뢰형
- "공식 안내 기준으로는"
- "현재 화면 기준으로 보면"
- "결제 경로에 따라 달라집니다"
- "정책은 바뀔 수 있어서 마지막에 한 번 더 확인하는 게 좋아요"

### 예외 인정형
- "대부분 이렇게 하면 돼요"
- "가끔 다르게 보일 수 있어요"
- "브라우저마다 조금씩 달라요"
- "그래도 안 되면 이 부분을 확인해보세요"

## 문장 예시

### 나쁜 예시
> 먼저 Microsoft Clarity 공식 사이트에 접속하여 로그인 버튼을 클릭해주세요. 로그인이 완료되면 대시보드로 이동하게 됩니다.

### 좋은 예시
> 먼저 Clarity 사이트에 들어가보세요. 로그인하면 대시보드가 바로 보입니다.

### 더 좋은 예시
> 먼저 Clarity 사이트에 들어가서 로그인합니다. 처음이면 프로젝트를 만들라는 화면이 먼저 나올 수 있어요.

## 참고: 네이버 블로그 포맷과 혼용

이 스타일 가이드는 기존 네이버 블로그 템플릿과 함께 사용한다.

- 문단 길이: 1~2문장 중심
- 말투: 따라하기 쉬운 블로그 톤
- 강조: 핵심 문장 노란 하이라이트, 위험/삭제 단어 빨간색
- 이미지: 제목 아래 대표 1장 + 단계별 주요 화면
- CTA: 본문 흐름을 끊지 않게 중간 이후 배치

네이버 포맷과 충돌하면 이 스타일 가이드를 우선한다.  
단, 검색 의도와 정보 정확도를 해치는 방식으로 자연스러움을 우선하지 않는다.
